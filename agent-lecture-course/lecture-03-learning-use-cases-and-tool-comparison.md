# 第 3 讲教案：让 Agent 帮助学习，顺便学会选工具

## 一、课程信息

- 课时：2 小时
- 主题：应用、比较与边界
- 核心目标：让听众学会把 Agent 放进真实学习流程，并建立基本选型与风险意识

## 二、教学目标

### 认知目标

- 理解 Agent 在学习场景中的适用边界
- 理解不同 OpenClaw 类工具的主要差异
- 理解“效率提升”和“思考外包”不是一回事

### 技能目标

- 能把一个学习任务拆分为适合 Agent 参与的环节
- 能用简单标准比较不同类型工具
- 能识别高风险或不恰当用法

## 三、讲前准备

### 教师准备

- 2 到 3 个学习场景案例
- 一张工具对比矩阵
- 一组风险判断题

### 推荐场景

- 英语阅读材料整理
- 作文结构检查与修改建议
- 错题归类与复盘
- 学习计划提醒与待办生成
- 资料检索与阶段性总结

## 四、时间安排

| 环节 | 内容 | 时间 |
|------|------|------|
| 1 | 开场：会用不等于乱用 | 15 分钟 |
| 2 | 学习场景中的正确位置 | 30 分钟 |
| 3 | 案例演示：英语与资料整理 | 30 分钟 |

| 6 | 全课总结 | 5 分钟 |

## 五、详细教学流程

### 环节 1：开场（15 分钟）

#### 开场话术

“到第三讲，真正的问题已经不是‘它厉不厉害’，  
而是‘它厉害归厉害，我到底该把它放在哪个位置上’。”

“如果把 Agent 用得好，它像一个助教。  
如果把 Agent 用歪了，它也可能像一个把你作业写得飞起、但考试时一个字都帮不上你的热心网友。”

### 环节 2：学习场景中的正确位置（30 分钟）

#### 适合 Agent 的任务

- 整理阅读重点
- 提取关键词和例句
- 初步检查作文结构
- 汇总材料
- 归类错题
- 制定提醒和待办

#### 不适合直接外包的任务

- 原创观点形成
- 独立思考训练
- 考试中的现场表达
- 价值判断与立场选择

#### 核心讲法

把学习过程拆成三层：

1. 收集整理
2. 理解消化
3. 输出表达

Agent 最适合帮忙的是第 1 层和第 1.5 层，  
也就是整理、提炼、辅助检查。  
真正决定你有没有学会的，还是第 2 层和第 3 层。

#### 金句

“Agent 能帮你把桌子收拾干净，  
但不能替你把知识长进脑子里。”

### 环节 3：案例演示（30 分钟）

#### 案例一：英语阅读辅助

任务：

“请把这篇文章整理成三部分：主旨、关键词、复习问题。”

观察点：

- 它是否抓住主旨
- 词汇提取是否有用
- 复习问题是否真的帮助理解

#### 案例二：作文修改辅助

任务：

“请检查这篇作文的结构、逻辑和语法问题，并给出修改建议，但不要直接整篇重写。”

讲解重点：

- 这属于“辅助修改”，不是“代写”
- 好的 Agent 应帮助你看见问题，而不是替你躲开问题

#### 案例三：错题归类

任务：

“请把这些错题按错误原因分类，并给出复习建议。”

#### 案例四：资料收集与整理

任务：

“请收集某个主题的 3 份入门资料，整理为一份适合复习的清单，并标注每份资料适合先看还是后看。”

讲解重点：

- 这是“辅助检索 + 辅助整理”
- 最终是否可信、是否适合自己，仍要人工判断

#### 讲师点拨

“如果一个工具让你越来越省时间，但也越来越不动脑，那你需要警惕。  
如果它让你把重复劳动省下来，把脑力集中到理解和表达上，那它就用对了。”

### 环节 4：OpenClaw 类工具比较（25 分钟）

#### 推荐分类法

不要死记产品名，先学会分类。

1. 原生开源型  
特点：自由度高、可定制、门槛高

2. 平台托管型  
特点：上手快、部署省心、限制较多

3. 场景集成型  
特点：嵌入现有平台、使用方便、能力边界受平台约束

#### 可举例但不必展开的平台

- 原生开源型：OpenClaw 原生自建
- 平台托管型：Kimi 等托管式方案
- 场景集成型：扣子、飞书等与既有平台深度结合的方案

#### 对比维度

- 上手门槛
- 自由度
- 隐私可控性
- 成本
- 适合人群
- 是否方便接入日常工作流

#### 幽默比喻

“选 Agent 工具，有点像选交通工具。  
自行车最自由，但得自己蹬。  
出租车最省心，但路线和费用未必最理想。  
地铁最稳定，但你得沿着它给你的线路走。”

#### 建议结论

- 只是初次体验：优先低门槛方案
- 想深度折腾：再考虑原生开源方案
- 想融入日常工作流：优先看平台集成能力

#### 可补充的一条进阶判断

如果一个工具最大的卖点是“看起来什么都能做”，那要多问一句：  
它靠什么做到，代价是什么，出错时我能不能看懂它错在哪。

### 环节 5：风险、边界与课堂讨论（15 分钟）

#### 必讲边界

- AI 结果可能错
- 引用和事实可能不准
- 权限使用可能带来误操作
- 隐私数据不能随意交出去
- 学术诚信不能被“效率”偷走
- 成本和额度也属于现实边界

#### 建议互动题

以下行为是否合理？

- 用 Agent 整理阅读材料
- 用 Agent 给作文提修改建议
- 用 Agent 直接代写并提交
- 用 Agent 汇总错题类型
- 用 Agent 替自己完成考试复习思考

#### 收束话术

“真正成熟的使用方式，不是把 Agent 供起来，  
也不是把它妖魔化，  
而是知道它该坐哪一张椅子。”

### 环节 6：全课总结（5 分钟）

#### 总结要点

- Agent 的关键价值是从“问答”走向“执行”
- 会部署不代表会使用，会使用不代表会判断
- 真正重要的是知道：什么时候该用，怎么用，用到哪一步就该停

#### 全课收尾金句

“AI 最好的位置，不是替你成长，  
而是把那些机械、重复、可拆解的活先接过去，  
把更重要的理解、判断和表达还给你自己。”

## 六、可直接上屏的收尾文案

工具的上限，决定它能帮你走多快。  
你的判断力，决定它会不会把你带偏。
