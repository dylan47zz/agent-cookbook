## 三、讲前准备建议

正式讲之前，建议先做这几件事：

- 从头滚动一遍网页，熟悉每个章节的位置
- 点一遍所有可交互区域，避免现场第一次点时自己先愣住
- 打开右下角“讲师备注”，看一遍每讲的备注提示
- 如果要投屏，提前全屏，避免浏览器标签栏分散注意力

## 四、页面结构速记

整套网页分为 5 个主要区域：

- 总览
- 第 1 讲：认识 Agent
- 第 2 讲：部署与配置
- 第 3 讲：学习应用与工具比较
- 结语

顶部导航可以直接跳转到任意章节。  
页面顶部进度条会显示整体讲述进度。  
右下角“讲师备注”会根据你当前所在章节自动切换内容。

## 五、推荐讲法

### 1. 总览页怎么讲

这一页不要停太久，建议控制在 3 到 5 分钟。

你的目标不是解释所有细节，而是先把整场的主问题立住：

“今天我们不是来看又一个聊天框，  
我们是来看 AI 为什么开始从一个答题工具，  
变成一个更像系统、助手和执行者的东西。”

这一页建议操作：

- 先不急着往下滚
- 先讲主标题和副标题
- 然后点“开始探索”或直接滚到地图区
- 用三张卡片快速概括三讲结构

这一页的关键词：

- 从问答到执行
- 从工具到系统
- 从新鲜感到判断力

### 2. 第 1 讲怎么讲

这一讲建议控制在 35 到 45 分钟。

建议讲法顺序：

1. 先读开场问题  
“如果我说，帮我整理文章、提炼重点、生成复习提纲，聊天 AI 和 Agent 会有什么不同？”

2. 然后点击“聊天 AI 会怎么做 / Agent 会怎么做”切换  
这里是第一讲最重要的开场交互。  
切换时不要只念文字，要顺手说一句：
“一个更像顾问，一个更像助理。”

3. 接着讲四个差异卡片  
这里适合边指边讲：
- 被动响应
- 目标驱动
- 封闭功能
- Skill 扩展

4. 再点“大脑 / 手脚 / 记忆 / 渠道 / 钥匙”  
建议重点讲“手脚”和“钥匙”，因为这是最容易让人一下子懂的地方。

5. 然后点流程链路  
这一段建议逐步点亮，不要一口气全讲完。  
点一步，讲一句，像带大家过机关。

6. 最后翻判断题卡片  
这部分很适合互动，可以让现场先猜，再点开答案。

这一讲最推荐重复的金句：

“我们不是在讨论谁更聪明，  
我们在讨论谁除了会说，还多长出了手和脚。”

### 3. 第 2 讲怎么讲

这一讲建议控制在 40 到 50 分钟。

讲法重点不是“逐项教程”，而是“让大家明白配置项在控制什么”。

建议讲法顺序：

1. 先看三种部署路径  
这里适合用“装修房子 / 精装公寓 / 酒店”那个比喻。

2. 再讲课堂推荐方案  
顺手强调：
“课堂里最重要的不是最硬核，而是先让它跑起来，再看懂为什么能跑。”

3. 然后点配置项卡片  
建议顺序是：
- 模型
- 工具
- Skill
- 权限
- 记忆
- 渠道

这里最值得停顿的是：
- Skill：因为很多人会第一次理解“能力包”这个概念
- 权限：因为这是风险教育的关键

4. 再点部署观察步骤  
这部分建议按顺序逐步点，让大家知道：
“第一次看 Agent 工作，别先被礼貌回复骗了，要先看它有没有真的动起来。”

5. 最后讲故障与风险  
这里不要讲得太吓人，但一定要稳稳落一句：
“能执行，不代表永远执行得对。”

这一讲最推荐重复的金句：

“配置不是装饰品。  
你每点开一个开关，其实都在定义它的能力边界和风险边界。”

### 4. 第 3 讲怎么讲

这一讲建议控制在 35 到 45 分钟。

它的目标不是列工具，而是教会判断。

建议讲法顺序：

1. 先看学习场景卡片  
从英语阅读、作文修改、错题归类、资料整理开始讲。  
这部分要不断强调一句话：
“适合交给 Agent 的，往往是流程；不能外包的，往往是成长。”

2. 再讲学习层次  
这一块要把“收集整理 / 辅助检查 / 理解表达”三层区分开。  
这会是整套课里最能帮助大家建立边界感的一段。

3. 然后讲工具分类  
这里不要先念产品名。  
先讲三类：
- 原生开源型
- 平台托管型
- 场景集成型

4. 再点选型矩阵筛选按钮  
这里最适合边点边讲：
- 门槛
- 自由度
- 隐私
- 集成能力

如果你讲到某个平台，可以顺手带一嘴，但不要被工具名绑架。

5. 最后翻边界判断卡  
这部分适合用来收束全课。

这一讲最推荐重复的金句：

“工具的上限决定它能帮你走多快，  
你的判断力决定它会不会把你带偏。”

### 5. 结语页怎么讲

这一页建议 5 分钟左右。

不要说成“谢谢观看”，而要说成“旅程收束”。

建议讲法：

- 逐条带过 5 个判断
- 不必展开太多
- 每一条都像落锤一样简洁

最后一句建议直接说：

“AI 最好的位置，不是替你成长，  
而是把那些机械、重复、可拆解的活先接过去，  
把真正重要的理解、判断和表达还给你自己。”

## 六、现场互动建议

如果现场气氛比较活，可以在这几个地方停下来互动：

- 第 1 讲开头：让大家先猜聊天 AI 和 Agent 的区别
- 第 1 讲结尾：先让大家判断哪些任务适合交给 Agent
- 第 2 讲中段：问大家觉得“权限”为什么危险
- 第 3 讲中段：让大家自己说一个学习任务，判断是否适合 Agent 参与
- 第 3 讲结尾：让大家说出自己最可能踩的一个坑

## 七、演讲节奏建议

如果整套课按 3 次讲座进行，每次 2 小时，可以参考这个节奏：

- 第 1 讲：概念 + 对比 + 判断
- 第 2 讲：部署 + 配置 + 风险
- 第 3 讲：应用 + 工具比较 + 边界

如果你是压缩成一场综合分享，也可以这样讲：

- 总览：5 分钟
- 第 1 讲精华：20 分钟
- 第 2 讲精华：20 分钟
- 第 3 讲精华：20 分钟
- 结语与问答：10 到 15 分钟

## 八、最后提醒

这套网页最适合的讲法，不是“念页面”，而是“借页面讲故事”。

你真正要带走观众的，不是某个平台的名字，  
而是这 3 件事：

- Agent 为什么值得理解
- Agent 为什么不能乱用
- Agent 怎样才能变成助手，而不是幻觉放大器
