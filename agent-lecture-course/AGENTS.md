# AGENTS.md

## 项目定位

这是一个围绕 `OpenClaw / Agent 入门讲座` 的课程项目目录。

它不是通用软件产品仓库，而是一个 `课程内容 + 演示网页 + 讲师使用资产` 的组合项目。

当前目标不是把功能做重，而是持续提高这套讲座的：

- 内容完整度
- 演示可讲性
- 页面可视化质量
- 后续接力维护效率

## 项目核心原则

### 1. 课程优先于炫技

任何改动都要先问自己：

- 这能不能帮助讲师讲得更顺？
- 这能不能帮助听众更快理解？
- 这是不是会让页面更像“演示工具”而不是“前端作品集”？

如果只是炫效果，但不提升讲解效率，优先不做。

### 2. 页面是讲义，不是海报

这个项目里的网页不只是好看，而是要承担真实讲解任务。

所以页面内容必须：

- 有结构
- 有节奏
- 有停顿点
- 有可互动点
- 有讲师可控的演示顺序

### 3. 参考资料可以借，但不能照搬

当前课程参考了以下资料：

- `Openclaw从入门到精通.md`
- `feishu-doc-P6zsdsgYco6i4XxLeIccvlpvnQe.md`
- 课程旧稿与官方文档

引用这些资料时要做两件事：

- 提炼成适合课堂表达的内容
- 区分官方口径、社区经验、第三方教程

不要把不同来源混讲成同一级别的事实。

### 4. 保持“能继续接力”

这个项目未来很可能会在不同电脑、不同会话、不同 Agent 之间继续推进。

所以改动时要优先保证：

- 文件命名清楚
- 结构稳定
- handoff 信息持续可用
- 页面里的新增模块不是临时拼贴

## 推荐 Agent 分工

如果多个 Agent 或多人协作，推荐按下面的思路分工。

### Agent A：课程内容 Agent

负责：

- 教案内容扩写
- 课程结构调整
- 讲师话术补充
- 参考资料提炼

不优先负责：

- 前端视觉细节
- JS 动画或交互修复

### Agent B：演示网页 Agent

负责：

- `site/index.html`
- `site/styles.css`
- `site/main.js`
- 页面视觉和交互优化

不优先负责：

- 大段课程文案重写
- 参考资料理解和结论提炼

### Agent C：课程运营/交付 Agent

负责：

- `演讲使用说明.md`
- `handoff.md`
- 截图替换计划
- 演示流程、现场操作建议

不优先负责：

- 课程理论结构重写
- 网页底层实现

## 修改网页时的约束

### 内容层面

- 不要把页面做成密密麻麻的大段文档墙
- 允许内容增厚，但必须分块、可扫读、可投屏
- 每新增一个知识块，最好同时考虑：
  - 标题
  - 核心结论
  - 可讲例子
  - 是否需要交互

### 视觉层面

- 保持当前“受《王国之泪》启发”的方向
- 借气质，不复刻游戏 UI
- 避免做成泛科技蓝紫色模板

### 交互层面

- 所有交互必须“点一次就懂”
- 不要引入复杂状态机
- 不要为了动画而动画
- 投屏稳定性优先于花哨效果

## 修改课程内容时的约束

- 保持“认知 + 部署 + 应用边界”三段主线
- 不要把课程带偏成纯安装教程
- 不要把课程带偏成纯平台营销介绍
- 不要默认听众熟悉编程或系统配置
