# 第 1 讲教案：Agent 是什么，为什么它不只是 ChatGPT

## 一、课程信息

- 课时：2 小时
- 主题：从“会回答”到“会做事”
- 核心目标：让听众建立对 Agent 的第一层认知，不再把它理解成“更贵一点的聊天框”

## 二、教学目标

### 认知目标

- 理解聊天 AI 与 Agent 的本质区别
- 理解 Agent 的基本组成与工作流程
- 理解什么任务适合交给 Agent，什么任务不适合

### 课堂产出

- 能用自己的话解释“Agent 为什么不是普通聊天 AI 的换皮版”
- 能列举 3 个适合 Agent 的任务和 2 个不适合 Agent 的任务

## 三、讲前准备

### 教师准备

- 一组对比演示案例：同一任务分别让聊天 AI 和 Agent 完成
- 一页 Agent 结构图：大脑、手脚、记忆、渠道、权限
- 一组课堂判断题

### 建议演示任务

- “整理一篇英语文章的重点并生成复习提纲”
- “把一周作业要求整理成待办清单”

## 四、时间安排

| 环节 | 内容 | 时间 |
|------|------|------|
| 1 | 开场破题：AI 只是聊天吗 | 15 分钟 |
| 2 | 聊天 AI vs Agent 核心对比 | 25 分钟 |
| 3 | Agent 的结构与流程 | 25 分钟 |
| 4 | 案例演示与拆解 | 25 分钟 |
| 5 | 任务分类互动 | 20 分钟 |
| 6 | 总结与提问 | 10 分钟 |

## 五、详细教学流程

### 环节 1：开场破题（15 分钟）

#### 目标

先把大家脑子里的旧印象拎出来，再轻轻打碎。

#### 建议开场话术

“很多人第一次听到 Agent，会以为它只是 ChatGPT 多了几个按钮。  
这就像看到电饭煲和电磁炉都能做饭，于是得出一个结论：它们差不多。  
听起来很合理，实际上很危险。”

“聊天 AI 最擅长的是一件事：你问，它答。  
Agent 想做的是另一件事：你给目标，它去办。”

#### 板书/页面关键词

- 问答
- 目标
- 规划
- 工具
- 执行

#### 提问互动

请大家快速回答：

- “帮我解释一个单词”像聊天 AI 任务，还是 Agent 任务？
- “帮我整理一篇文章、提炼重点、生成复习提纲”更像哪一类？

#### 讲师提示

这里不要急着下定义，先让大家感受到“任务复杂度”和“动作链条”的不同。

### 环节 2：聊天 AI vs Agent 核心对比（25 分钟）

#### 核心讲点

可以把聊天 AI 看作“会说话的顾问”，把 Agent 看作“能动手的助理”。

聊天 AI 的强项：

- 理解问题
- 生成回答
- 解释概念
- 给建议

Agent 的新增能力：

- 接收目标而不是只接收问题
- 主动拆解步骤
- 调用外部工具
- 对环境执行操作
- 把多步动作串成闭环

#### 可补充的现实背景

可以补一句：  
OpenClaw 这一轮受到关注，不只是因为“回答更强”，而是因为很多人第一次比较直观地看到 AI 从“坐在聊天框里回话”，变成了“能接任务、能调工具、能把一整件事做下去”。

#### 幽默比喻

“聊天 AI 像班里那个理论特别好、嘴也很快的同学。  
你问他题，他能讲半小时。  
但如果你说‘来，别讲了，你去把实验器材领回来、把分组名单整理好、再把结果发群里’，他大概率会看着你。”

“Agent 则更像班委。  
不一定每一句都讲得天花乱坠，但他真的会去跑腿、安排、执行，还会回来跟你说：这事我办到哪一步了。”

#### 建议对比表

| 对比项 | 聊天 AI | Agent |
|------|------|------|
| 核心能力 | 回答问题 | 完成任务 |
| 工作方式 | 一问一答 | 目标驱动 |
| 工具使用 | 通常较弱或受限 | 较强，能串联多工具 |
| 执行动作 | 少 | 多 |
| 风险 | 幻觉、误答 | 幻觉、误答、误操作 |

#### 可直接上屏的四个差异关键词

- 从“被动响应”到“目标驱动”
- 从“按需打开”到“常驻待命”
- 从“封闭功能”到“Skill 扩展”
- 从“单一入口”到“跨平台接入”

#### 过渡话术

“所以我们不是在讨论谁更聪明。  
我们在讨论的是：谁除了会说，还多长出了手和脚。”

### 环节 3：Agent 的结构与流程（25 分钟）

#### 核心结构

用一套固定比喻讲清：

- 模型：大脑，负责理解和规划
- 工具：手脚，负责实际操作
- 记忆：备忘录，负责记住上下文和偏好
- 渠道：门和电话，负责和人沟通
- 权限：钥匙，决定它能进哪些房间

#### 讲解文案

“如果只有大脑，没有手脚，那它就只能当军师。  
如果有手脚没有钥匙，那它也只能站在门口叹气。  
如果钥匙给太多，它又可能一不小心把家里锅都端走。  
所以 Agent 的威力，来自‘大脑 + 手脚 + 钥匙 + 记忆’的组合。”

#### 流程讲解

Agent 的典型流程：

1. 接收目标
2. 理解任务
3. 拆分步骤
4. 调用工具
5. 观察结果
6. 调整动作
7. 返回结果

#### 讲师提示

这里强调：Agent 的关键不是“一次回答更长”，而是“会在过程里行动”。  
也可以顺手埋个伏笔：第二讲会看到，这种“会行动”的能力，背后依赖的不只是模型，而是模型、工具、权限、渠道一起工作。

### 环节 4：案例演示与拆解（25 分钟）

#### 推荐案例

任务：  
“请整理这篇英语文章的主要观点，提取 8 个核心词汇，并生成一份 5 分钟复习清单。”

#### 演示方式

先展示聊天 AI 的表现，再展示 Agent 的表现。

聊天 AI 可能会：

- 直接给出总结
- 生成词汇表
- 但通常不会自己组织文件、持续跟踪、进一步动作

Agent 可能会：

- 读取内容
- 分类信息
- 输出结构化结果
- 进一步生成复习清单或待办

#### 拆解重点

演示后请观众回答：

- 它只是“说得更完整”吗？
- 还是已经出现了“处理流程”？
- 哪一步最体现 Agent 的价值？

#### 讲师点拨

“如果聊天 AI 像一个口头答题王，  
那 Agent 更像一个会自己去翻书、找资料、列步骤、交作业草稿的助手。”

### 环节 5：任务分类互动（20 分钟）

#### 互动题目

请判断以下任务更适合交给哪一类工具：

- 解释一个语法点
- 归纳错题成因
- 代写整篇作文直接提交
- 定时提醒复习计划
- 根据一份材料生成提纲
- 替你完成考试中的思考

#### 标准引导

适合 Agent 的任务通常具备三个特点：

- 目标明确
- 步骤可拆
- 结果可检查

不适合直接交给 Agent 的任务通常涉及：

- 价值判断
- 原创表达
- 深度思考训练
- 高风险不可逆操作

#### 课堂金句

“能交给 Agent 的，往往是流程。  
不能外包给 Agent 的，往往是成长。”

### 环节 6：总结与提问（10 分钟）

#### 本讲总结

- Agent 不是聊天 AI 的皮肤包
- Agent 的关键差异是会规划、会调工具、会执行
- Agent 适合规则明确、步骤清晰的任务
- Agent 不能代替真正的理解和思考

#### 收束话术

“第一讲如果你只记住一句话，请记这句：  
聊天 AI 更像一个会回答的人，Agent 更像一个会干活的系统。”

## 六、可直接上屏的收尾文案

Agent 的价值，不在于替你思考，  
而在于帮你把那些可以被拆解、被执行、被复查的任务处理得更省力。
