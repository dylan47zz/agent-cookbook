# 第 2 讲教案：把 Agent 跑起来，理解部署与配置

## 一、课程信息

- 课时：2 小时
- 主题：从“听懂概念”到“真的跑起来”
- 核心目标：让听众完成一次基础部署或配置，并理解配置项背后的含义

## 二、教学目标

### 认知目标

- 理解不同部署方式的差异
- 理解模型、工具、权限、记忆、渠道等配置项的作用
- 理解为什么 Agent 的能力越强，风险也越高

### 技能目标

- 跑通一次基础部署或平台创建流程
- 能完成最基础的配置
- 能执行一个简单任务并观察过程

## 三、讲前准备

### 课堂策略

本讲建议采用“双层方案”：

- 主方案：低门槛、一键部署或平台托管版，保证多数人能成功
- 演示方案：教师展示原生 OpenClaw 或更完整配置流程，帮助大家建立全景认知

### 教师准备

- 一个已经可用的 OpenClaw/同类 Agent 环境
- 一套课堂实操截图
- 一份常见问题清单
- 一个简单演示任务

### 推荐演示任务

- “帮我整理今天的学习任务并生成待办清单”
- “读取一段英文材料，输出重点与词汇表”

## 五、详细教学流程

### 环节 1：复盘与导入（10 分钟）

#### 导入话术

“上一讲我们搞明白了 Agent 是什么。  
这一讲轮到真正的灵魂拷问了：说了半天，它到底怎么跑起来？”

“理解概念像是看菜谱。  
部署和配置，就是第一次真的开火。  
这时候你会发现：锅、火、调料、食材，缺一个都不行。”

### 环节 2：三种部署路径讲解（20 分钟）

#### 三种常见路径

1. 本地部署  
自己搭环境，自己配模型，自己管权限。  
优点是自由度高、隐私强，缺点是门槛高。

2. 平台托管  
平台帮你准备大部分环境。  
优点是省心，缺点是更受平台限制。

3. 一键部署  
最适合初次体验。  
优点是上手快，缺点是可定制空间有限。

#### 可结合的现实案例

- 原生本地型：自己部署 OpenClaw，本地接模型、工具和权限
- 平台托管型：例如 Kimi 这类提供云端环境的方案
- 场景接入型：例如扣子、飞书这类把 Agent 接进既有平台的方案

这里建议重点讲“类型”，产品名只作为例子，避免课程被单个平台绑住。

#### 建议讲法

不要把这三种路径讲成“谁更高级”，而要讲成“谁更适合当前目标”。

#### 幽默比喻

“本地部署像自己装修房子，插座安在哪儿都能自己决定，但你得先会看图纸。  
平台托管像住精装修公寓，拎包入住，改造自由有限。  
一键部署像住酒店，最快能住进去，但别指望把墙打掉重装。”

### 环节 3：课堂实操（50 分钟）

#### 实操目标

让大家至少完成以下动作中的大部分：

- 创建一个可用 Agent
- 选择模型或默认运行环境
- 完成基础权限确认
- 了解可用工具或 Skill
- 执行一条简单任务指令

#### 推荐实操步骤

1. 打开部署入口
2. 创建或启动 Agent
3. 选择模型/套餐/默认配置
4. 查看可用工具或 Skill
5. 下达第一条任务
6. 观察执行过程
7. 记录成功点与卡点

#### 实操观察点

- 它有没有真的开始执行，而不只是礼貌回复
- 它调用了哪些工具或能力
- 它是否能在当前平台里完成一个小闭环
- 它出错时更像是卡在模型、权限、工具还是渠道上

#### 课堂提醒

- 有人成功很快，有人会卡住，这很正常
- 实操课最怕的不是报错，是不敢报错
- 一旦卡住，先看哪一步没对，不要先怀疑人生

#### 可用讲师话术

“如果现在有人已经成功跑起来了，不要得意得太早。  
Agent 跑起来只是刚拿到驾照，还没上高速。”

“如果有人卡住了，也别急。  
技术世界的一条铁律是：第一次配置不出点问题，反而像哪里不太正常。”

### 环节 4：配置项背后的逻辑（20 分钟）

#### 要解释的配置项

- 模型
- 工具
- 权限
- 记忆
- 渠道
- Skill

#### 讲法建议

不要只说“这里填什么”，要解释“它决定什么”。

#### 讲解文案

模型：决定它怎么思考，像给大脑选发动机。  
工具：决定它能做哪些动作，像给它配手和脚。  
权限：决定它能进哪些地方，像给它发钥匙。  
记忆：决定它能记住多少上下文，像给它配备忘录。  
渠道：决定你从哪里叫它，像决定你是打电话、发消息还是当面喊它。  
Skill：决定它额外会哪些专门本事，像给它报培训班后新增技能。

#### 关于 Skill 的讲法

可以把 Skill 讲成“Agent 的可插拔能力包”。  
它有点像手机上的 App，也有点像游戏里的装备。  
区别在于：App 往往是你主动点开用，Skill 更像装到 Agent 身上后，它会在合适的时候自己调用。

#### 重点提醒

“配置不是装饰品。  
你每点开一个开关，实际上都在决定它的能力边界和风险边界。”

### 环节 5：常见问题与排障（15 分钟）

#### 常见问题

- 部署成功了，但不会响应
- 能聊天，但不会执行动作
- 指令发下去，结果跑偏
- 上下文越来越乱
- 权限不够或权限太多
- Skill 装了，但不会正确调用

#### 排障顺序

1. 先看环境是否真的启动
2. 再看模型或账号是否可用
3. 再看工具/Skill 是否可调用
4. 再看权限是否合理
5. 最后再优化提示词

#### 课堂金句

“很多人一出问题就开始疯狂改提示词。  
但有时候问题根本不在脑子，而在手脚没装好，或者钥匙没发对。”

### 环节 6：安全提醒与总结（5 分钟）

#### 必讲风险

- 不要轻易给高权限
- 不要在重要设备或重要目录里乱试
- 不要把含糊目标直接交给强执行能力 Agent
- 不要默认它每一步都理解对了
- 不要被“一键部署”四个字骗得以为就完全没有代价

#### 可补充的现实提醒

一些平台方案虽然更省心，但可能会带来会员费用、额度限制、模型限制或平台约束。  
所以课堂上最好反复强调：部署变简单了，不等于选择变简单了。

#### 收束话术

“Agent 最迷人的地方，是它终于不像个只会回消息的人。  
Agent 最吓人的地方，也是它终于不像个只会回消息的人。”

## 六、本讲总结

- 部署方式没有绝对高低，只有适不适合
- 配置项本质上是在定义 Agent 的能力边界
- 真正会用 Agent 的人，会看它怎么做，不只看它做出了什么
- 权限越大，越要谨慎

## 七、可直接上屏的收尾文案

配置一个 Agent，  
本质上是在回答一个问题：  
我要让它帮我做什么，又绝不允许它乱做什么？
